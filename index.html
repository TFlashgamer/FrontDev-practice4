Вступ

JavaScript - це кросплатформна, об'єктно-орієнтована мова сценаріїв. Це невелика і легка мова. Всередині хост-середовища (наприклад, веб-браузера) JavaScript може бути підключений до об'єктів його середовища для забезпечення програмного контролю над ними.

JavaScript містить стандартну бібліотеку об'єктів, таких як масив, дата та математика, та основний набір мовних елементів, таких як оператори, структури управління та оператори. Основний JavaScript можна розширити для різних цілей, доповнивши його додатковими об'єктами; наприклад:

  • На стороні клієнта JavaScript розширює основну мову, надаючи об'єкти для керування браузером та його моделлю документа DOM (DOM). Наприклад, розширення на стороні клієнта дозволяють програмі розміщувати елементи у формі HTML та реагувати на події користувача, такі як клацання миші, введення форми та навігація по сторінці.
  • На стороні сервера JavaScript розширює основну мову, надаючи об’єкти, важливі для запуску JavaScript на сервері. Наприклад, розширення на стороні сервера дозволяють додатку спілкуватися з базою даних, забезпечувати безперервність інформації від однієї виклику до іншої програми або здійснювати маніпуляції файлами на сервері.

JavaScript і Java

JavaScript та Java схожі в деяких аспектах, але принципово відрізняються в деяких інших. Мова JavaScript нагадує Java, але не має статичної типізації Java та чіткої перевірки типу. JavaScript слідує більшості синтаксису вираження Java, умовам іменування та основних конструкцій потоку управління, що стало причиною того, що він був перейменований з LiveScript у JavaScript.

На відміну від системи збирання класів у часі Java, побудованої деклараціями, JavaScript підтримує систему виконання на основі невеликої кількості типів даних, що представляють числові, булеві та рядкові значення. У JavaScript є об'єктна модель на основі прототипу замість більш поширеної об'єктної моделі на основі класу. Модель на основі прототипу забезпечує динамічне успадкування; тобто те, що передається у спадок, може змінюватись для окремих об'єктів. JavaScript також підтримує функції без особливих декларативних вимог. Функції можуть бути властивостями об’єктів, виконуючись як типово типізовані методи. На відміну від системи збирання часу класів Java, побудованої деклараціями, JavaScript підтримує систему виконання на основі невеликої кількості типів даних, що представляють числові, булеві та рядкові значення. JavaScript - це дуже вільна мова порівняно з Java. Вам не потрібно оголошувати всі змінні, класи та методи. Вам не потрібно перейматися тим, чи є методи публічними, приватними чи захищеними, і вам не потрібно реалізовувати інтерфейси. Змінні, параметри та типи повернення функцій не вводяться явно.

Змінні

Ви використовуєте змінні як символьні імена для значень у вашій програмі. Назви змінних, що називаються ідентифікаторами, відповідають певним правилам.

Ідентифікатор JavaScript повинен починатися з літери, підкреслення (_) або знака долара ($); наступні символи також можуть бути цифрами (0-9). Оскільки JavaScript відрізняється великими літерами, літери включають символи "А" до "Z" (великі регістри) та символи "а" через "z" (малі літери).

Ви можете використовувати літери ISO 8859-1 або Unicode, такі як å і ü в ідентифікаторах. Ви також можете використовувати послідовності втечі Unicode як символи в ідентифікаторах. Деякі приклади легальних імен - Number_hits, temp99 та _name.

Оголошення змінних

Ви можете оголосити змінну трьома способами:

З ключовим словом var. Наприклад,

var x = 42 Цей синтаксис можна використовувати для оголошення локальних та глобальних змінних.

Просто присвоївши йому значення. Наприклад,

x = 42 Це завжди оголошує глобальну змінну. Він генерує суворе попередження JavaScript. Не слід використовувати цей варіант.

З ключовим словом нехай. Наприклад,

let y = 13 Цей синтаксис можна використовувати для оголошення локальної змінної області дії блоку. Див. Область змінної області нижче.

Змінна область застосування

Коли ви оголошуєте змінну за межами будь-якої функції, вона називається глобальною змінною, оскільки вона доступна для будь-якого іншого коду в поточному документі. Коли ви оголошуєте змінну в межах функції, вона називається локальною змінною, оскільки вона доступна лише в межах цієї функції.

JavaScript перед ECMAScript 2015 не має області оператора блоку; швидше, змінна, оголошена в блоці, локальна для функції (або глобальної області), в якій знаходиться блок. Наприклад, наступний код буде входити в протокол 5, оскільки сфера x - це функція (або глобальний контекст), в рамках якої оголошено x, а не блок, який у цьому випадку є оператором if

if (true) { var x = 5; } console.log(x); // 5 Ця поведінка змінюється при використанні декларації про дозвіл, введеної в ECMAScript 2015. if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined

Глобальні змінні

Глобальні змінні насправді є властивостями глобального об'єкта. На веб-сторінках глобальним об’єктом є вікно, тому ви можете встановити та отримати доступ до глобальних змінних за допомогою синтаксису window.variable.

Отже, ви можете отримати доступ до глобальних змінних, оголошених в одному вікні або кадрі, з іншого вікна або кадру, вказавши ім'я вікна або кадру. Наприклад, якщо в документі оголошено змінну під назвою phoneNumber, ви можете посилатися на цю змінну з iframe як parent.phoneNumber.

Константи

Ви можете створити лише для читання, названий константою за допомогою ключового слова const. Синтаксис постійного ідентифікатора такий же, як і для змінного ідентифікатора: він повинен починатися з літери, підкреслення або знака долара і може містити алфавітні, числові або підкреслювальні символи. const PI = 3.14; Константа не може змінювати значення за допомогою призначення або бути повторно оголошеним під час запуску сценарію. Це має бути ініціалізовано до значення.

Правила діапазону для констант такі ж, як і для змінних областей пропускного блоку. Якщо ключове слово const пропущено, ідентифікатор передбачається представляти змінну.

Ви не можете оголосити константу з тим самим іменем як функцію чи змінну в тій самій області. Наприклад:

// THIS WILL CAUSE AN ERROR function f() {}; const f = 5; // THIS WILL CAUSE AN ERROR ALSO function f() { const g = 5; var g; //statements } Однак атрибути об'єкта не захищені, тому наступне твердження виконується без проблем. const MY_OBJECT = {"key": "value"}; MY_OBJECT.key = "otherValue";

Типи даних

Останній стандарт ECMAScript визначає сім типів даних:

  • Шість типів даних, які є примітивними:

    • Булева. справжнє і хибне.
    • нуль. Спеціальне ключове слово, що позначає нульове значення. Оскільки JavaScript відрізняється від регістру, null не є тим самим, як Null, NULL або будь-який інший варіант.
    • невизначений. Властивість верхнього рівня, значення якої не визначено.
    • Номер. 42 або 3,14159.
    • Рядок. "Хороший"
    • Символ (новий у ECMAScript 2015). Тип даних, екземпляри яких унікальні та незмінні.

if ... else твердження

Використовуйте оператор if для виконання оператора, якщо логічна умова істинна. Використовуйте необов'язковий пункт else для виконання оператора, якщо умова хибна. Оператор if виглядає так: if (condition) { statement_1; } else { statement_2; } умовою може бути будь-який вираз, який оцінює як істинне чи хибне. Дивіться Boolean для пояснення того, що оцінюється як істинне і хибне. Якщо умова оцінюється як істинна, оператор_1 виконується; в іншому випадку заява_2 виконується. statement_1 та statement_2 можуть бути будь-якими твердженнями, включаючи подальші вкладені, якщо заяви.

Ви також можете скласти висловлювання, використовуючи else, якщо у вас послідовно перевіряли кілька умов:

if (condition_1) { statement_1; } else if (condition_2) { statement_2; } else if (condition_n) { statement_n; } else { statement_last; } У випадку декількох умов буде виконуватися лише перша логічна умова, яка оцінюється як істинна. Щоб виконати кілька операторів, згрупуйте їх в операторі блоку ({...}). Загалом, це найкраща практика завжди використовувати висловлювання блоків, особливо під час введення, якщо заяви: if (condition) { statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else { statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; } Доцільно не використовувати прості завдання в умовному виразі, оскільки це завдання можна переплутати з рівністю при огляді коду. Наприклад, не використовуйте наступний код: if (x = y) { /* statements here */ } Якщо вам потрібно використовувати завдання в умовному виразі, поширеною практикою є розміщення додаткових дужок навколо призначення. Наприклад: if ((x = y)) { /* statements here */ }

while твердження

У той час як оператор виконує свої твердження до тих пір, поки зазначена умова оцінюється як істинна. Час твердження виглядає так: while (condition) statement Якщо умова стає хибною, оператор у циклі припиняє виконання, а керування переходить до оператора, що слідує за циклом.

Перевірка стану відбувається перед виконанням оператора в циклі. Якщо умова повертається як істина, оператор виконується і умова перевіряється знову. Якщо умова повертається помилково, виконання зупиняється і контроль передається в оператор, наступний час.

Щоб виконати кілька операторів, використовуйте оператор блоку ({...}) для групування цих операторів.

Приклад:

Наступне, а цикл повторюється до тих пір, поки n становить менше трьох:

var n = 0; var x = 0; while (n < 3) { n++; x +=n; } З кожною ітерацією цикл збільшується n і додає це значення до x. Тому x і n приймають такі значення:
  • Після першого проходу: n = 1 і x = 1
  • Після другого проходу: n = 2 і x = 3
  • Після третього проходу: n = 3 і x = 6

Після завершення третього проходу умова n < 3 вже не відповідає дійсності, тому цикл припиняється.

Оголошення функцій

Визначення функції (також називається декларацією функції або операційною функцією) складається з ключового слова функції з наступним:
  • Назва функції.
  • Список аргументів функції, укладений у дужки та розділений комами.
  • Виписки JavaScript, що визначають функцію, укладені у фігурні дужки, {}.

Наприклад, наступний код визначає просту функцію з назвою квадрат:

function square(number) { return number * number; } Функціональний квадрат бере один аргумент, який називається числом. Функція складається з одного твердження, яке говорить повернути аргумент функції (тобто число), помножену на себе. Оператор return вказує значення, повернене функцією. return number * number; Примітивні параметри (наприклад, число) передаються функціям за значенням; значення передається функції, але якщо функція змінює значення параметра, ця зміна не відображається глобально або у викличній функції.
